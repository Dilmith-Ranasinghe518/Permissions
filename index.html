Parent Pal – Why we request permissions

Parent Pal uses device permissions only to provide child safety and parental supervision features. We request permissions when you use a feature that needs them.

Privacy note: Parent Pal does not sell personal data. Permission access is used for app features and safety alerts only. You can change permissions anytime in Android Settings.

Live Child Location / Safety Zones

Location
ACCESS_FINE_LOCATION ACCESS_COARSE_LOCATION ACCESS_BACKGROUND_LOCATION (if enabled)

Why we need it:

  • Show the child device location to the parent for safety.
  • Enable location-based alerts (e.g., safe zone entry/exit) when the parent turns it on.
  • Background location is used only for zone alerts when the app is not open (if you enable that feature).

Child Safety Notifications & Alerts

Notifications
POST_NOTIFICATIONS (Android 13+) Notification Listener Access

Why we need it:

  • Send safety-related alerts and reminders from the child device.
  • Allow parents to view child-related alert notifications for supervision (not personal message content).

Full-Screen Event / Emergency Alert Display

Display over other apps
SYSTEM_ALERT_WINDOW

Why we need it:

  • Show important safety or event alerts on top of other apps so they aren’t missed.
  • Used only for child alerts/reminders that you enable, not for ads.

Scheduled Reminders & Event Alerts

Exact alarms
SCHEDULE_EXACT_ALARM

Why we need it:

  • Trigger reminders and safety alerts at the exact time you set.
  • Helps alerts work reliably even when the app is closed.

Parental Controls (Device Protection)

Device admin
Device Administrator (DPM)

Why we need it:

  • Enable parental control protections like preventing unauthorized changes on the child device.
  • Used only for parental control and child safety features you activate.

App Usage Monitoring

Usage access
PACKAGE_USAGE_STATS (Usage Access)

Why we need it:

  • Show how much time the child spends on apps for parental insight.
  • Help enforce limits or routines (if you enable them).

On-device Supervision Controls (Optional)

Accessibility (only if used)
Accessibility Service

Why we need it:

  • Support supervision features like blocking restricted actions or helping enforce limits (if enabled).
  • Not used to read typed passwords or personal chats.

If your app does not use Accessibility features, remove this section.

Reliable Safety Features While App Runs

Foreground service
FOREGROUND_SERVICE FOREGROUND_SERVICE_LOCATION (if used)

Why we need it:

  • Keep child safety features running reliably when needed (with a visible system notification).
  • Used for active safety functions, not for background advertising.
Need help?
You can manage permissions in Settings → Apps → Parent Pal → Permissions. If a permission is denied, some safety features may not work as expected.
